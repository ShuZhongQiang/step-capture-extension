# 2026-04-10 工作日志

## Bug 修复 - getActiveTab 三重 fallback

- 问题：侧边栏打开时焦点窗口是扩展窗口，`lastFocusedWindow:true` 拿不到用户网页 tab
- 修复：改用三重策略 fallback（1.聚焦窗口active tab → 2.所有窗口active tab → 3.最近访问的网页tab），`isWebTab()` 过滤允许 `http://`、`https://`、`file://`


- 根本原因：侧边栏页面调用 `chrome.tabs.query({ active:true, currentWindow:true })` 会拿到错误的 tab（侧边栏不属于任何普通 tab），导致 startRecording 传错 tabId，content.js 收不到消息
- 修复方案：
  1. `manifest.json` 补充 `"tabs"` 权限
  2. `background.js` 新增 `getActiveTab` 消息处理，在 Service Worker 侧用 `lastFocusedWindow:true` 查询 tab，并过滤 chrome:// 等非普通页面
  3. `popup.js` 新增 `getActiveTab()` 函数，委托 background 获取 tab，`startRecording` / `stopRecording` 全部改用此函数
